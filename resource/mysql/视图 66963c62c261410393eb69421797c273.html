视图

定义视图

  • 创建视图
    • 基于单表的视图
      #建立计算机科学系学生的视图#
mysql> create view v_s 
    -> as       
    -> select sno,sname,sage from s where sdept='cs';
Query OK, 0 rows affected (0.01 sec)
    • 基于多表的视图
      #建立选修了1号课程的学生视图(包括学号、姓名、成绩)#
mysql> create view v_s1(sno,sname,grade) 
    -> as 
    -> select s.sno,sname,grade from s,sc where s.sno=sc.sno and sc.cno='1';
Query OK, 0 rows affected (0.02 sec)
    • 基于视图的视图
      #立选修了1号课程且成绩在90分以上的学生的视图#
mysql> create view v_s2
    -> as
    -> select sno,sname,grade from v_s1 where grade>=90;
Query OK, 0 rows affected (0.02 sec)
    • 带表达式的视图
      • 利用表达式来设置派生属性列
      #定义一个反映学生出生年份的视图#
mysql> create view v_s3(sno,sname,sbirth)
    -> as
    -> select sno,sname,2005-sage from s;     
Query OK, 0 rows affected (0.02 sec)
    • 建立分组视图
      #将学生的学号及他的平均成绩定义为一个视图#
mysql> create view v_s4(sno,gavg)
    -> as
    -> select sno,avg(grade) from sc group by sno;  
Query OK, 0 rows affected (0.02 sec)
    • 注意事项
  • 删除视图
    • 示例
      drop view is_s1;

查询视图

  • 查询方法（针对用户）
    • 查询视图与查询基本表一致
  • 查询方法（针对DBMS）
    • 视图实体化法
    • 视图消解法（有缺陷，故DBMS会限制部分查询）

更新视图

  • 更新方法（针对用户）
    • 更新视图与更新基本表一致
  • 更新方法（针对DBMS）
    • 视图实体化法
    • 视图消解法
  • 注意
    • 指定 with check option 子句后，DBMS会在更新视图时进行检查，防止用户通过视图对不属于视图范围内的基本表数据进行更新
    • 部分视图是不可更新的（两种更新方法均存在这种情况），因为其无法唯一地有意义地转换成相应基本表的更新

视图的作用

  • 视图可以简化用户的操作
    • 多表查询、嵌套查询、派生表查询
  • 视图使用户能以多种角度看待同一数据
  • 视图对重构数据库提供了一定程度的逻辑独立性

with check option 子句

  • 在定义的视图上进行查询、插入、更新和删除操作
    • 查询：当DBMS采用视图实体化法时，可以进行查询；当DBMS采用视图消解法时，查询存在限制。
    • 插入：有with check option子句时，要保证插入后，数据可以被视图查询出来；无with check option子句时，不存在限制。
    • 更新：有with check optio子句时，要保证更新后，数据可以被视图查询出来；无with check option子句时，不存在限制。
    • 删除：不存在限制。
  • WITH CHECK OPTION语句的作用。
    • 使用with check option创建视图时，将对视图上的所有更新和插入语句进行检查以确保新行符合视图的where条件。如果不符合，则该操作将引发错误并被拒绝。
    • 例子：定义视图v1时添加where条件sage > 20，当插入一行数据时，如果存在with check option子句，则必须保证该行的sage > 20，否则将引发错误。
