DDL常用命令

  • 表管理
    • 创建库
      • 推荐写法：create database [库名];
      • create database if not exists [库名];
    • 删除库
      • 推荐写法：drop database if exists [库名];
      • drop database [库名];
    • 创建表
      create table [表名]
(
[字段名] [类型(宽度)] [约束条件] [comment ‘注释’],
[字段名] [类型(宽度)] [约束条件] [comment ‘注释’],
[字段名] [类型(宽度)] [约束条件] [comment ‘注释’]
);
      约束说明：
      • not null：标识字段不能为空（可以为空字符串’’或者空格）
      • default value：设置字段默认值，默认为value
      • primary key：标识字段为该表的主键，可以唯一的标识记录，支持多字段，插入重复的会报错
        • 写法1：跟在列后 create table test (a int primary key);
        • 写法2：在所有列定义之后定义
          create table test1
(
a int, 
b char(5),
primary key(a)
);
          create table test2
(
a int, 
b char(5),
c varchar(10),
primary key(a,b)
);
      • foreign key：为表中的字段设置外键
        • 语法： foreign key ([当前表的列名]) references [被参照表]([引用的外键表的列名]);
        • 写法：
          create table test
(
a int,
b char(5),
c char(5),
foreign key(b) references test(c)
);
      • unique key：标识该字段的值唯一，支持多字段，插入重复的会报错
        • 写法1：跟在字段后
          create table test (a int not null unique key);
        • 写法2：在所有列定义之后定义
          create table test1
(
a int not null,
unique key(a)
);
          create table test2
(
a int, 
b char(5),
c varchar(10),
unique key(a,b)
);
      • auto_increment：标识该字段的值自动增长（整数类型，且为主键）
        自动增长： 自动增长默认从1开始，每次+1 自动增长的初始值和步长都能在mysql中进行设置
    • 删除表
      • 推荐写法：drop table if exists [表名];
      • drop table [表名];
    • 修改表名
      • alter table [旧表名] rename to [新表名];
      • alter table [旧表名] rename [新表名];
    • 表设置备注
      • alter table [表名] comment ‘[备注信息]’;
    • 复制表
      • 只复制表结构
        • create table [表名] like [被复制的表名];
      • 复制表结构+数据
        • create table [表名] as select [字段],… from [被复制的表名] where [条件];
        • create table [表名] (select ...);
      • 只复制表数据
        • insert into [表名] (select ...)
  • 表中列的管理
    • 添加列
      • alter table [表名] add column [列名] [类型] [约束];
      • alter table [表名] add column [列名] [类型] [约束] first/after [列名];
    • 添加约束
      • alter table [表名] add [表级完整性约束];
      • 主键约束： alter table [表名] add constraint [约束名称] primary key ([列名]);
      • 唯一键约束： alter table [表名] add constraint [约束名称] unique key ([列名]);
      • 外键约束： alter table [表名] add constraint [约束名称] foreign key ([列名]) references [被引用的表名]([被引用的列名])
    • 修改列
      • 修改类型 alter table [表名] modify column [列名] [新类型] [约束] first/after [列名];
      • 修改列名和类型： alter table [表名] change column [列名] [新列名] [新类型] [约束];
      • 修改列名： alter table [表名] rename column [列名] to [新列名];
      • 修改表名： alter table [表名] rename to/as [新表名];
      • 修改列排序： alter table [表名] order by [列名]…
    • 默认值
      • 设置默认值： alter table [表名] alter column [列名] set default [值];
      • 删除默认值： alter table [表名] alter column [列名] drop default [值];
    • 删除列
      • alter table [表名] drop column [列名];
    • 删除约束
      • 删除主键： alter table [表名] drop primary key;
      • 删除外键： alter table [表名] drop foreign key [外键约束名]; drop index [外键索引名称] on [表名];

    • 创建索引
      • create index [索引名] on [表名];
    • 显示索引
      • show index from [表名];
    • 修改索引
      • alter index [索引名] rename to [新索引名]
    • 删除索引
      • drop index [索引名] on [表名];

    • 索引的意义

      MySQL 数据库有 2 中检索方式：

      • 全表扫描（效率较低）

        查询 ename=’KING’ ： select * from emp where ename=’KING’; 如果 ename 没有添加索引，那么通过 ename 过滤数据的时候，ename 字段会全表扫描

      • 通过索引检索（提高查询效率）

        创建索引的情况：

        1. 该字段的数据量庞大
        1. 该字段很少使用 DML 操作（索引需要维护，DML 操作太多的时候，影响检索效率）
        1. 该字段经常出现在筛选条件 where 中实际开发中根据项目需求或客户需求综合调整
