二分查找

整数二分

//区间[begin, end]被划分成[begin, mid]和[mid + 1, end]时使用
int bsearch_1(int begin, int end)
{
	while (begin < end)
	{
		int mid = begin + end >> 1;
		if (check(mid))end = mid;
		else begin = mid + 1;
	}
	return begin;
}

//区间[begin, end]被划分成[begin, mid - 1]和[mid, end]时使用
int bsearch_2(int begin, int end)
{
	while (begin < end)
	{
		int mid = begin + end + 1 >> 1;
		if (check(mid))begin = mid;
		else end = mid - 1;
	}
	return begin;
}

二分搜索

bool binary_search(const int n, const int x, int ar[])
{
	int start = 0;
	int end = n - 1;
	while (start <= end)
	{
		int i = (start + end) / 2;
		if (ar[i] == x)
		{
			return true;
		}
		else if (ar[i] > x)
		{
			end = i - 1;
		}
		else
		{
			start = i + 1;
		}
	}
	return false;
}

结论(假设数列有n个元素)：

  • start = 0，更新语句为start = i + 1（整数相除向下取整）
  • end = n - 1，更新语句为end = i - 1

分析：

证明条件中不应为end > 0，假设存在end > 0条件：

(0 + 9) / 2 = 4.5 → 4 > x end = 4 - 1 = 3 (0 + 3) / 2 = 1.5 → 1 > x end = 1 - 1 = 0 不符合end > 0的条件，故不存在 矛盾

证明条件中不必有end ≥ 0：

条件只需包含start ≤ end即可，因为若所需数值比数列中的所有数都小，start = i + 1不可能被执行，故start = 0恒成立，此时end ≥ 0与start ≤ end意义相同

证明更新语句中不应为start = i，假设更新语句为start = i：

(0 + 9) / 2 = 4.5 → 4 < x start = 4 (4 + 9) / 2 = 6.5 → 6 < x start = 6 (6 + 9) / 2 = 7.5 → 7 < x start = 7 (7 + 9) / 2 = 8 < x start = 8 (8 + 9) / 2 = 8.5 → 8 < x start = 8 此时陷入死循环，start无法更新成9（因为两个Int类型的数相除的商向下取整，结果始终为8）

假设现有一包含10个元素的数列：

证明更新语句中不应为end = i，假设更新语句为end = i：

此时假设x = 4 (0 + 9) / 2 = 4.5 → 4 对应 9 > x end = 4 (0 + 4) / 2 = 2 对应 5 > x end = 2 (0 + 2) / 2 = 1 对应 3 < x start = 0 + 1 = 1 (1 + 2) / 2 = 1.5 → 1 对应 3 < x start = 1 + 1 = 2 (2 + 2) / 2 = 2 对应 5 > x end = 2 此时陷入死循环，end始终等于start，程序找不到与所需数值相对应的数值，但仍符合循环条件

实践：

(0 + 9) / 2 = 4.5 → 4 > x end = 4 - 1 = 3 (0 + 3) / 2 = 1.5 → 1 = x 数列中存在一个数值与所需数值相同

(0 + 9) / 2 = 4.5 → 4 > x end = 4 - 1 = 3 (0 + 3) / 2 = 1.5 → 1 > x end = 1 - 1 = 0 (0 + 0) / 2 = 0 > x end = 0 - 1 = -1 不符合start ≤ end的条件，故不存在

(0 + 9) / 2 = 4.5 → 4 < x start = 4 + 1 = 5 (5 + 9) / 2 = 7 < x start = 7 + 1 = 8 (8 + 9) / 2 = 8.5 → 8 < x start = 8 + 1 = 9 (9 +9) / 2 = 9 < x start = 9 + 1 = 10 不符合start ≤ end的条件，故不存在

存在其他的初始化与更新语句，但这些初始化必须与相应的更新语句对应
